TIPOS DE DECLARACIONES

En programacion, las declaraciones indican que a partir del momento en que declaramos algo, cuando usemos una palabra o un nombre, ésta deberá poder contener un valor o una acción;

Dicho de otra manera; cuando en JavaScript querramos declarar un lugar dónde tengamos localizados una série de datos o valores, haremos

  var lugarDondeGuardoUnValor;

Nota
si nos fijamos, se utiliza una palabra reservada (var) para indicar que la palabra que la sigue, tendrá un uso específico (almacenar valores, que podrán ser por supuesto consultados a posteriori, mediante esa misma palabra);

Y en el caso de que querramos declarar una acción, podremos usar la siguiente notación:

  function accionQuePintaUnaSuperficie(superficie, color);

Nota
en éste último ejemplo, la palabra reservada que se usa es function (que es un sinónimo de acción o cálculo). Y en esta función o acción, se necesita conocer la superfície a ser pintada y el color elegido, es por ello, que se expresan en el interior del paréntesis que sigue al nombre de la acción.

IMPORTANTE
Para expresar o referir a una variable necesitaremos básicamente usar su “nombre”, mientras que para expresar o referir una acción deberemos siempre abrir y cerrar paréntesis a continuación de su nombre (independientemente de si la acción necesita que le sea proporcionada alguna acción de partida para poderse llevar a cabo).

TIPOS DE INSTRUCCIONES

Principalmente se distinguen tres tipos de instrucciones, según su propósito general:

  • Instrucciones de entrada: cuyo propósito es obtener unos datos (de una fuente ajena como un fichero de nuestro sistema, o una dirección remota) para ser posteriormente procesados.

    ajax.load(url);

    Ejemplo en que se lanza una acción llamada “load” (cargar contenido) de una dirección de internet (url) que contiene un contenido (compuesto por datos).

    Nota
    Las intrucciones de entrada, consisten siempre en la ejecución de una acción (function) que tiene por propósito obtener unos datos.
    Nombres de funciones como .read(source), son indicativos de que nos encontramos ante una instrucción de entrada de datos

  • Instrucciones de salida: cuyo propósito es justamente el opuesto a las de entrada, y por lo tanto, serán instrucciones que escriban o retornen datos, tal como si de una fuente de datos misma se tratara.

    ajax.post(url,content);

    Ejemplo en el que se lanza una acción llamada “post” (publicar contenido) a una dirección de internet (url) que recibirá el contenido a publicar (compuesto por datos).

    Nota:
    Existen acciones (funciones) cuyo nombre es indicativo de que se trata de instrucciones de escritura, salida (o envio de datos) como .log(contenido), .write(contenido), .put(url,contenido), .send(address,contenido), .print(contenido), exit(), return [variable].

  • Instrucciones de proceso: cuyo propósito es utilizar datos obtenidos mediante instrucciones de entrada (que se han dado en el mismo procedimiento o en procedimientos previos), para finalmente facilitar uno o varios resultados, ya sea a subsiguientes instrucciones (por lo general de salida, aunque pueden ser de cualquiera de los 3 tipos), del mismo proceso o de procesos posteriores.

    decodedUrl = decodeURI(encodedUrl);

    Ejemplo en el que se lanza una acción llamada “decodeURI” (decodificar una “dirección” que conduce a un recurso con contenido) .

    Nota:
    Las instrucciones de proceso, no necesariamente ejecutan una acción (función), pues tambien pueden tratarse de operaciones aritméticas, lógicas o de manipulación de texto, como por ejemplo:
    nombre_completo = nombre + ' ' + apellidos

TIPOS DE INSTRUCCIONES DE TIPO PROCESO

Las instrucciones de proceso a su vez se distinguen según el propósito específico de las mismas. Debemos asumirlas en base a sus equivalencias con las formas presentes en los diagramas de flujo.

  • Instrucciones de declaracion (o declarativas): són aquellas que anuncian una declaración (ver TIPOS DE DECLARACIONES), y por tanto reservan un espacio en la memoria para almacenar datos o la descripción de una rutina (o función).
    En Javascript se emplea la palabra (reservada) var para indicarlo.

    var variable_indicando_una_posición_de_memoria_para_guardar_datos;

  • Instrucciones de asignación: en las que se asigna un valor a una variable, o dicho de otro modo, se escribe un valor a un espacio de memoria reservado para almacenar datos.
    En Javascript se emplea el operador = para indicarlo.

    variable_indicando_una_posición_de_memoria_para_guardar_datos = valor;

  • Instrucciones de cómputo (o de cálculo): en las que se realiza una operación (aritmética, lógica o de manipulación de textos) para obtener un valor (que necesariamente será almacenado en una variable (mediante una asignación) o bién facilitado a una función
    En Javascript se emplean el operador + para unir dos textos como uno solo.

    prefijo_internacional+numero_de_telefono_nacional;

    En Javascript se puede usar una instrucción de cálculo en lugar de un valor, o una variable, en la invocación de funciones.

    realizarLlamadaTelefónica( prefijo_internacional+numero_de_telefono_nacional );
  • Instrucciones compuestas o complejas: en las que se realizan varias instrucciones como si de una sola se tratara.
    porcentajeVotosAFormaciones = (recontarVotosUrnas() + recontarVotosPorCorreo() - recontarVotosNulos()) * 100 / cantidadElectorado;

    NOTA: En la expresión anterior se usan paréntesis para agrupar las operaciones aritméticas de suma y resta, para resolverlas previamente a realizar las operaciones de multiplicación y división (que tiene mayor precedencia), exactamente igual que sucede con la notación matemática:

%VotosPartidos=(VotosUrnas+VotosCorreo−VotosNulos)⋅100Electorado

  • Instrucciones de control (o SENTENCIAS): Aunque no son en sí mismas instrucciones compuestas, permiten determinar la ejecución o no, de procesos o de instrucciones. Todas ellas, tienen un inicio y un fin. En todas ellas, en el inicio se expresa una evaluación de un valor.
    Como es imaginable el valor evaluado puede ser, perfectamente, el resultado de una instrucción de cómputo, ya sea simple o compuesta.
    Dentro de las intrucciones de control encontramos:

    • Sentencias condicionales: Que permiten ejecutar conjuntos de instrucciones, en función del cumplimiento de una condición. La condición es una evaluación de un valor en términos BOOLEANOS, es decir, que sólo se interpretará el valor como algo CIERTO o FALSO.
      Las sentencias condicionales, necesariamente determinan una desviación en un flujo de ejecución de las instrucciones de un programa en el caso de que se cumpla la condición, que es justamente cuando el valor se evalua como CIERTO.
      Las sentencias condicionales, optativamente pueden determinar otra desviación específica de un flujo de ejecución para el caso de que una condición no se cumpla, o lo que es lo mismo, cuando el valor se evalua como FALSO.

       // ↓ Inicio de una sentencia condicional (principal) expresada por "if"
 // ↓ que implicitamente indica una desviación en el flujo de ejecución
 // ↓ cuando la condición (hace_buen_dia y no_tengo_compromisos) se cumple
 if (hace_buen_dia && no_tengo_compromisos) {

     // ↓ Conjunto de instrucciones que se ejecutarán en la desviación
     // ↓ del flujo de ejecución cuando la condición se cumple
     salir_a_pasear();
     volver_a_casa();

 } // ← Fin del tratamiento en caso de cumplimiento de la condición

 // ↓ inicio del tratamiento para cuando la condición no se cumple,
 // ↓ expresado por "else"
 else {

     // ↓ Conjunto de instrucciónes que se ejecutarán
     // ↓ en la desviación del flujo
     if(hace_buen_dia){ // ← Inicio de una sentencia condicional secundaria
         atender_compromisos(); // ← conjunto de instrucciones (solo una)
     } // ← Fin de la sentencia condicional de segundo nivel, que no tiene
       // ↑ ningún tratamiento específico para el incumplimiento

 } // ← Fin del tratamiento en caso de incumplimiento
   // ↑ de la condición evaluada en la sentencia condicional principal

 // ↓ Continuación del flujo de ejecución no controlado
 // ↓ por sentencias condicionales
 leer_un_libro();

    • Sentencias alternativas: Se distinguen de las sentencias condicionales, por cuando permiten evaluar el valor de un modo distingo al Booleano, y por lo tanto, ofrecen tratamientos específicos según el valor evaluado.
      Lo que plantean las sentencias alternativas equivaldria a disponer sentencias condicionales anidadas cada una en el interior del tratamiento para el caso de incumplimiento de la sentencia condicional anterior; y además, todas tendrían que fijar condiciones distintas en torno a la misma variable o cómputo.

      if (letra == 'a'){...} // desviación del flujo A cuando la letra és igual a 'a'
else{
  if (letra == 'b'){...} // desviación del flujo B cuando la letra és igual a 'b'
  else{}
    ...
    else{
       if (letra == 'z'){...} // desviación del flujo Z cuando la letra és igual a 'z'
       else{ // desviación optativa y alternativa a las anteriores
           informar_letra_no_es_del_abecedario_latin();
       }
    }
  }
}

      la sentencia alternativa equivalente al anterior serie de anidamientos, sería:

      switch (letra) {
case 'a' : // desviación del flujo A cuando el valor de letra es 'a'
    ...
    break;
case 'b' : // desviación del flujo B cuando el valor de letra es 'b'
    ...
    break;
...
case 'z' : // desviación del flujo Z cuando el valor de letra es 'z'
    ...
    break;
default: // desviación optativa y alternativa a las anteriores
    informar_letra_no_es_del_abecedario_latin();
    break;
}

    • Sentencias iterativas: Determinan la repetición de un conjunto de instrucciones mientras se cumpla una condición (se evalue como CIERTO en términos booleanos)
      Supongamos que se pretende recoger los exámenes repartidos entre un conjunto de alumnado una vez cada alumno/a haya terminado su exámen, repetidamente, hasta que se hayan recogido todos.
      La manera cómo se expresaría usando sentencias que no fueran la iterativa, seria:

      dar_por_iniciada(sesion);
repartir(examenes,alumnado);
// ↓ Inicio de sentencia condicional primaria
if (quedan_examenes_por_recoger()){

    // ↓ Inicio de bloque de instrucciones si la condición se cumple
    esperar(tiempo);
    if (hay_algun_examen_terminado()){
        recoger(examen);
    }

    // ↓ Repetición de sentencia condicional primaria
    // ↓ dentro del bloque de instrucciones cuando la condición se cumple
    if (quedan_examenes_por_recoger()){
        esperar(tiempo);
        if (hay_algun_examen_terminado()){
            recoger(examen);
        }

        // ↓ Nueva repetición de sentencia condicional primaria
        // ↓ dentro del bloque de instrucciones cuando la condición se cumple
        if (quedan_examenes_por_recoger()){
            esperar(tiempo);
            if (hay_algun_examen_terminado()){
                recoger(examen);
            }

            // ↓ Otra nueva repetición de sentencia condicional primaria
            // ↓ dentro del bloque de instrucciones cuando la condición se cumple
            if (quedan_examenes_por_recoger()){
                ... //repetición del mismo bloque de instrucciones
            }
        }
    }
}

// ↓ Instrucción siguiente en el flujo principal
// ↓ que se ejecuta una vez se incumple la condición
dar_por_finalizada(sesion);

      El caso anterior, exigiría tantas evaluaciones de condición como entregas pendientes queden. Eso significa, implícitamente, que LA CONDICIÓN EVALUADA, PUEDE HABER CAMBIADO SU VALOR RESPECTO A LAS ANTERIORES EVALUACIONES.
      Ese cambio, es señal inequívoca de que la iteración (repetición de un mismo bloque de instrucciónes), debe finalizar, Y ES, A SU VEZ, LA CONDICIÓN EXIGIBLE PARA EL ABANDONO DE UNA ITERACIÓN: QUE EL VALOR DE LA CONDICIÓN PUEDA CAMBIAR EN ALGUN MOMENTO MIENTRAS SE ITERA.
      Si experesáramos el ejemplo anterior usando una sentencia iterativa, la condición seria evaluada nuevamente, una vez se hubiera ejecutado el bloque de instrucciones encerrado en la sentencia iterativa. El resultado sería el siguiente:

      dar_por_iniciada(sesion);
repartir(examenes);

// ↓ Inicio de una sentencia iterativa expresada por "while"
// ↓ La condición evaluada es la misma que en la sentencia condicional
while (quedan_examenes_por_recoger()){

    // ↓ Mismo bloque de instrucciones cuando la condición se cumple
    esperar(tiempo);
    if (hay_algun_examen_terminado()){
        recoger(examen);
    }
}  // ← Fin de la sentencia iterativa, indica que debe retrocederse
   // ↑ al punto dónde se inició y volver a evaluar la condición

// ↓ Instrucción siguiente en el flujo principal
// ↓ que se ejecuta una vez se incumple la condición
// ↓ y por tanto, ya no puede iterarse
dar_por_finalizada(sesion);

    • Sentencias iterativas cuantificadas: Le llamanos “cuantificadas” a las sentencias iterativas que realizan un recorrido, y que por tanto, en la evaluación de la condición, están evaluando si la posición de un indice es ya la última del recorrido. Los índices suelen ser variables declaradas a tal efecto, cuyo único propósito es almacenar el valor de una posición. El término “cuantificado” se aplica dado que para expresar la última posición en el recorrido, debe cuantificarse su valor en términos de distancia respecto al punto de partida.
      El punto de partida, será siempre el primer valor que se haya asignado a la variable empleada como índice.
      Las sentencias condicionales, esperan que a cada iteración (sin excepción), el índice haya avanzado su posición y se haya acercado al fin del recorrido.
      Un ejemplo para poner a punto un número conocido de ordenadores en una aula de estudio sería:

      // ↓ Bloque de declaracióens
var revisiones; // ← Declaración del índice, que indicará
                // ↑ el número de ordenadores revisados
var problema; // ← Declaración de una variable dónde se asignará
              // ↑ el problema detectado en el ordenador que se esté revisando

// ↓ Bloque de instrucciones
abrir_aula();
preparar_material();

// ↓ Inicio de una sentencia iterativa cuantificada expresada por "for"
// ↓ revisiones = 0 es la asignación PUNTO DE PARTIDA
// ↓ revisiones < NUMERO_DE_ORDENADORES es LA CONDICIÓN para seguir iterando
// ↓ incrementar1(revisiones) es la instrucción de cómputo del AVANCE
for (revisiones = 0; revisiones < NUMERO_DE_ORDENADORES; incrementar1(revisiones)){

  // ↓ Bloque de instrucciones a ejecutar si la condición se cumple
  if (hay_un_problema_en_el_siguiente_ordenador()){
      arreglar_siguiente_ordenador();
  }

} // ← Fin de la sentencia iterativa, indica que debe ejecutarse el AVANCE
  // ↑ y retrocederse al punto dónde se inició, para volver a evaluar la condición

// ↓ Continuación del flujo principal
recoger_material();
cerrar_aula();
